    SystemScript

    A simple low-level programming language for system programming and hardware manipulation

    SystemScript is a simple low-level programming language designed for direct hardware manipulation, system programming, and security research. It provides a unique blend of low-level access with high-level abstractions, allowing programmers to interact with hardware while maintaining a readable syntax.

    Key Features

    Direct Hardware Access

    • CPU Register Manipulation: Directly read and write CPU registers using the $ command
    • Memory-Mapped I/O: Access hardware devices through memory-mapped registers with the % command
    • Port I/O: Control hardware ports for legacy device communication using the # command
    • GPU Programming: Send commands directly to the GPU for hardware-accelerated operations

    Memory and Code Manipulation

    • Dynamic Memory Management: Allocate and free memory at runtime
    • Code Injection: Insert and modify executable code at runtime
    • JIT Compilation: Compile SystemScript code to native machine code on-the-fly
    • Process Manipulation: Create processes and inject code into existing processes

    Networking Capabilities

    • Raw Packet Crafting: Create network packets at the lowest level
    • Network Device Control: Directly manipulate network interface cards
    • Protocol Implementation: Build custom network protocols from scratch

    Security and System Control

    • Vulnerability Research: Tools for testing system security
    • System Call Interface: Execute privileged operations via system calls
    • Binary Patching: Modify running applications

    Command Reference

    Basic Operations

    Command Description Example
    + Increment value at pointer +
    - Decrement value at pointer -
    > Move pointer right >
    < Move pointer left <
    [ Start loop (if current value != 0) [>+<-]
    ] End loop [>+<-]
    . Output character at pointer .
    , Input character to pointer ,

    Memory Access

    Command Description Example
    @addr Set memory pointer to address @1000
    * Dereference pointer *
    =value Set value at pointer =42
    :label Define label :loop
    ;label Jump to label ;loop

    Hardware Control

    Command Description Example
    #port[=value] Read/write hardware port #0x3F8=65
    $reg[=value] Access CPU register $eax=42
    %addr[=value] Access memory-mapped I/O %0xF000=1
    !syscall Execute system call !0x80

    Bitwise Operations

    Command Description Example
    &value Bitwise AND &0x0F
    |value Bitwise OR |0xF0
    ^value Bitwise XOR ^0xFF
    ~ Bitwise NOT ~

    Example Programs

    Hello World

    SC"Hello, World!"SP

    This simple example copies the string "Hello, World!" to memory and then prints it.

    Hardware Port Blinking LED (x86)

    :loop
#0x378=1  // Turn on LED on parallel port (pin 2)
=100[>+<-]  // Delay
#0x378=0  // Turn off LED
=100[>+<-]  // Delay
;loop

    This example toggles an LED connected to a parallel port (commonly found in older systems). It creates a simple blinking pattern by writing values to hardware port 0x378.

    Memory Manipulation

    @1000          // Set pointer to address 1000
=72.>=101.>=108.>=108.>=111.> // Write "Hello"
@1000          // Go back to beginning 
[.>]           // Print until zero

    This example demonstrates basic memory manipulation by writing ASCII values for "Hello" to sequential memory locations and then printing them out.

    GPU Pixel Drawing Example

    // Map GPU framebuffer
M{0xF0000000,8388608,gpufb}
=0  // Store device ID at current pointer

// Set color (ARGB)
=255>
=0>
=0>
=255>

// Draw a pixel at (100,100)
@100+100*800*4  // Calculate pixel address (y*width*bpp + x*bpp)
%0=0xFF0000FF   // Write color to memory-mapped framebuffer

    This advanced example shows how to interact with a GPU framebuffer to draw pixels directly. It maps the GPU memory and then calculates the position to place a red pixel at coordinates (100,100).

    Network Packet Injection

    // Craft a simple Ethernet packet
N{eth0,FF,FF,FF,FF,FF,FF,00,11,22,33,44,55,08,00}

    This example sends a raw Ethernet packet to the network interface eth0. The packet contains a broadcast MAC address and an EtherType of 0x0800 (IPv4).

    Safety and Security Considerations

    SystemScript provides powerful capabilities that can affect system stability if used incorrectly. Exercise caution when working with hardware directly.

    • Privileged Operations: Many commands require elevated privileges (root/admin) to work with real hardware
    • System Stability: Writing to arbitrary memory or hardware registers can crash your system
    • Security Implications: Code injection and process manipulation should be used only in controlled environments

    For learning and experimentation, the interpreter defaults to a simulation mode that provides feedback without accessing real hardware.

    Building from Source

    # Standard build
gcc -o systemscript systemscript.c

# Build with real hardware access (use with caution)
gcc -o systemscript -DENABLE_REAL_HARDWARE -DENABLE_GPU_ACCESS -DENABLE_CODE_INJECTION systemscript.c

    Advanced Usage

    Memory-Mapped Device Registration

    To work with hardware devices through memory mapping:

    // Map a device (e.g., a GPU framebuffer)
M{0xF0000000,4096,gpufb}
// The device ID is stored at the current pointer
// Access the device with the % command
%0=255  // Write to the first byte of the mapped device

    JIT Compilation

    To improve performance for computationally intensive tasks:

    // Define a function to be JIT compiled
J{
  // Factorial calculation
  =1  // Initialize result to 1
  >
  =5  // Calculate factorial of 5
  [
    <*>  // Multiply result by counter
    -    // Decrement counter
  ]
  <.    // Output result
}
